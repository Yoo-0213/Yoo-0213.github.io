DDR的基本原理

DDR层次图

层次化由高到低分别为：
DDR -> 内存颗粒 -> Bank Group -> Bank -> Array -> Cell

这里举一个简单的例子进行解释：
一个 Array 有 216 行 和 210 列，那么这个 Array 的容量为 216 * 210 = 64 Mb
一个 Bank 有 8 个 Array ，那么这个 Bank 的容量为 8 * 64 Mb = 64 MB
一个 Bank Group 有 4 个 Bank ，那么这个 Bank Group的容量为 4 * 64 MB = 256 MB
一个内存颗粒有4个 Bank Group，那么这个内存颗粒的容量为 4 * 256 MB = 1 GB

Tip

我们常说的一个内存颗粒的位宽是多少，其实就是指一个 Bank 里面有多少个 Array

我们每次读写一个逻辑地址，其实就是同时读取 8 个内存颗粒（假设这个 DDR 有 8 个内存颗粒，每个内存颗粒位宽为 8 bit）中同一个位置 Bank Group 里的 Bank 的 8 个 Array 的同一行同一列，这样我们就获得了 8 * 8 = 64 bit 的数据

再配合 DDR 的"数据预取技术"，我们可以获得更多数据。如 DDR4 预取数据为 8n ，也就是读写一个逻辑地址，我们可以获得 8 * 8 * 8 = 512 bit 的数据

DDR技术
预取

首先，回顾一下内存颗粒的位宽概念。在一个 Array 中，行地址和列地址交叉选中单个 Cell，从中获取到 1 bit 数据。若我们将 2 个 Array 叠加在一起，就能获取到 2 bit数据；若是 8 个 Array 叠加在一起，就能获取到 8 bit数据，这就是前面提到的一个 Bank 里面有多少 Array ，内存颗粒的位宽就是多少

首先，回顾一下DRAM的读操作。DDR收到行激活命令时，Array 的对应行被激活，对应行的电容发生了轻微的电压变化而被感应放大器捕捉到，感应放大器则根据这电压变化判断存储的数据是 0 还是 1；接着收到列读取命令，

【FPGA】I2C协议
时序图 数据有效性 I2C 协议对数据的采样是发生在 SCL 为高电平期间，除了起始、停止信号，在数据传输 SCL 为高电平期间，SDA 必须保持稳定，不允许发生变化，只有在 SCL 为低电平时才可跳变 起始信号和停止信号 起始信号：在 SCL 为高电平时，SDA 由高变低 停止信号：在 SCL 为高电平时，SDA 由低变高 应答信号和非应答信号 应答信号 ：第 9 个 SCL 时钟周期，SDA 信号为低电平； 非应答信号：第 9 个 SCL 时钟周期，SDA 信号为高电平； Note 应答信号出现在主机发送 1 字节之后，即第 9 个 SCL 时钟周期。这时，主机释放 SDA 总线，把控制权交给从机，由于上拉电阻的原因，此时总线为高点平；如果从机能正确收到主机发来的数据，从机会把 SDA 总线拉低，表示应答信号
